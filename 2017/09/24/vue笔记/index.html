vue笔记

发表于 2017-09-24 | 分类于 工具 | | 阅读次数

vue是一个轻量级MVVM框架，数据驱动+组件化的前端开发，社区完善。

1.安装nodejs

直接下载安装，此处记录的是windows更新node、npm的笔记

2.安装生产项目

  • 如果浏览器打开之后，没有加载出页面，有可能是本地的 8080 端口被占用，需要修改一下配置文件 config>index.js>dev>port
  • build 的路径前缀修改为 ‘ ./ ‘（原本为 ‘ / ‘），因为打包之后，外部引入 js 和 css 文件时，如果路径以 ‘ / ‘ 开头，在本地是无法找到对应文件的（服务器上没问题）。所以如果需要在本地打开打包后的文件，就得修改文件路径。
  • vue-cli是vue的脚手架工具，包含目录结构、本地调试、代码部署、热加载、单元测试。

3.注意事项

  • 1.任何css、js不要写到index.html里，写这里的不会被遗弃封装。通用的写到app.vue里，最后所有css会被封装成一个css
  • 先定义路由，一般一次性定义好所有页面，也可以边写边定义
  • 同局域网设备可访问
    因webpack设置安全权限，只有localhost可以访问，所以同局域网内其他设备无法访问。
    vue -> config -> index.js 中的host部分改成host：’0.0.0.0’，同局域网设备即可访问。

4.Axios

保存全局数据

  • src下新建文件夹config里建dev.js 域名在此处改

5.打包

自己的项目文件都要放到 src 文件夹下

  • 打包完成后，会生成 dist 文件夹，如果已经修改了文件路径，可以直接打开本地文件查看
  • 项目上线时，只需要将 dist 文件夹放到服务器就行了。

5.Vue-router笔记

·vue路由

  • 防止url输入错误找不到页面：{ path: ‘*’,redirect:’/‘ }
  • 跳转到上一次浏览的页面：this.$router.go(-1);}
  • 制定跳转的地址：this.$router.replace(‘/menu’);}
  • 制定跳转路由的名字下：this.$router.replace({name:’menuLink’});}
  • 通过push进行跳转：this.$router.push(‘/menu’);}
  • 通过push跳转路由的名字：this.$router.push({name:’menuLink’});}
